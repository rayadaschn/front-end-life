## Angular 模块移植到 vue2

这个项目的难点主要有两个。
第一是 技术迁移，因为 Angular 和 Vue 在组件结构、依赖注入和数据流上差异比较大，不能简单复制代码，需要重新设计组件结构和数据逻辑。
第二是 协作难度，当时团队有多个项目并行，这个模块又被几个项目依赖，所以迁移过程中需要协调开发进度，避免影响其他项目。

当时主要做了三件事：
第一是先 分析原有 Angular 模块结构，把组件、服务层和公共逻辑梳理出来，再设计对应的 Vue2 实现方式，比如把 service 层拆成 API 请求层，把组件逻辑重新按 Vue 组件组织。

    原模块的组成:
    - 页面组件
    - 子组件
    - service 层
    - 数据流和依赖关系

第二是 分阶段迁移，先抽离公共逻辑，再逐步迁移组件，最后整体集成到 Vue 项目中，这样风险比较可控。
第三是 加强团队沟通和进度同步，提前确认依赖关系，保证接口和数据结构兼容。

最终这个模块顺利迁移到 Vue2 项目中，既保证了业务稳定，也统一了技术栈，后续的维护和开发效率都提升了。

## 如何做项目设计

第一是项目分层，比如 pages、modules、components、services、store 等，保证职责清晰。
第二是组件分层，区分 UI 组件、业务组件和页面组件，提高复用性。
第三是统一请求层和状态管理，避免组件直接请求接口，保证数据流清晰。
第四是完善工程化体系，比如 eslint、commit 规范、CI/CD 和测试体系。

如果项目规模更大，还可以考虑通过微前端拆分系统，提高团队协作效率。

## 如何设计一个前端组件库？

首先是组件设计原则，比如单一职责、高复用性和可配置性，组件尽量通过 props 和 slots 来控制行为。

在架构上我会采用组件模块化结构，每个组件独立目录，并抽离 hooks、utils 和 styles，方便维护和扩展。

样式方面会设计统一的 design token，例如 CSS variables，保证主题能力和样式一致性。

构建方面会使用 Rollup 或 Vite 打包，同时支持 ESM 和 CJS，并实现按需加载以减少包体积。

此外还会提供完整的 TypeScript 类型定义、组件文档系统（例如 Storybook 或 VitePress），以及单元测试保证组件稳定性。

最终通过 npm 发布并结合 semantic version 做版本管理。

## 遇到需求不合理怎么办？

如果遇到不太合理的需求，一般需要先理解产品的真实目标，然后再给出技术角度的建议。

先确认：

- 这个需求要解决的业务问题是什么
- 有没有更简单的实现方式
- 是否会对性能或者开发成本有比较大的影响

如果确实存在问题，需要把影响说明清楚，比如：

- 开发复杂度
- 用户体验
- 性能影响

然后给出替代方案，和产品一起讨论一个更合理的实现方式。
